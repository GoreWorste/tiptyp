# TipTyp

**Тренажёр слепой печати** — проверка скорости набора, учётные записи и личная статистика. Интерфейс на русском и английском, 100 тем оформления (50 тёмных и 50 светлых).

## Возможности

| Функция | Описание |
|--------|----------|
| **Тест на скорость** | Таймер от первого нажатия, подсветка правильных/ошибочных символов, WPM и точность в реальном времени |
| **Три режима слов** | Осмысленные слова из словаря, генерация n-граммной моделью, псевдослова из слогов |
| **Языки** | Русский и английский — переключатель в шапке, язык сохраняется при переходах |
| **Регистрация и вход** | Учётные записи, пароли хранятся в виде хеша (Werkzeug) |
| **Моя статистика** | История попыток, лучший и средний WPM для авторизованных пользователей |
| **Темы** | Отдельная страница с 50 тёмными и 50 светлыми темами, выбор сохраняется в браузере |
| **Адаптивная вёрстка** | Удобно на телефоне, планшете и 4K-мониторе |

Управление: **Tab** или **Ctrl+Enter** — перезапуск теста.

## Технологии

- **Backend:** Python 3.11, Flask 3, Flask-Login, Flask-SQLAlchemy
- **База данных:** SQLite (файл `tiptyp.db`)
- **Безопасность:** хеширование паролей (PBKDF2-SHA256)
- **Фронтенд:** HTML/CSS/JS, без сборки; CSS-переменные для тем
- **Генерация слов:** встроенный словарь, n-граммная модель (pickle), слоги

## Установка и запуск

### Требования

- Python 3.10+
- pip

### Шаги

1. Клонируйте репозиторий и перейдите в каталог:

   ```bash
   git clone https://github.com/GoreWorste/tiptyp
   cd tiptyp
   ```

2. Создайте виртуальное окружение (рекомендуется):

   ```bash
   python -m venv venv
   # Windows:
   venv\Scripts\activate
   # Linux/macOS:
   source venv/bin/activate
   ```

3. Установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

4. Запустите приложение:

   ```bash
   python app.py
   ```

5. Откройте в браузере: **http://127.0.0.1:5000**

При первом запуске создаётся база `tiptyp.db` и при необходимости обучается модель слов (файлы в `instance/`).

## Docker

Запуск в контейнере удобен для доступа с других устройств в локальной сети (телефон, планшет, другой ПК).

1. Установите [Docker](https://docs.docker.com/get-docker/) и [Docker Compose](https://docs.docker.com/compose/install/).

2. В корне проекта:

   ```bash
   docker compose up -d
   ```

3. Сайт доступен:
   - на этом ПК: **http://localhost:5000**
   - с других устройств в сети: **http://\<IP-вашего-ПК\>:5000**

Данные (БД `tiptyp.db` и папка `instance/`) сохраняются в **папке проекта** на хосте — те же файлы используются при запуске и через Docker, и через `python app.py`, поэтому пользователи и статистика общие. Остановка:

```bash
docker compose down
```

**С телефона не открывается?** Проверьте: тот же Wi‑Fi, в адресе используйте IP ПК (не localhost), откройте порт 5000 в брандмауэре. Подробно: [DOCKER_ACCESS.md](DOCKER_ACCESS.md).

## Структура проекта

```
tiptyp/
├── app.py              # Flask-приложение, маршруты, модели
├── word_generator.py   # Режимы генерации слов (словарь, модель, слоги)
├── word_model.py       # N-граммная модель для генерации слов
├── train_word_model.py # Скрипт переобучения модели
├── requirements.txt
├── Dockerfile
├── docker-compose.yml
├── templates/          # Jinja2-шаблоны
├── static/             # CSS, JS, темы
├── scripts/            # Генерация CSS тем (generate_themes.py)
└── instance/           # Создаётся при запуске: БД, модели (или volume в Docker)
```
